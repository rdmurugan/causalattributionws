General Questions

What is causal attribution?

Causal attribution uses advanced statistical methods to determine which marketing touchpoints actually caused a conversion, rather than just correlating with it. Unlike traditional attribution models that assign credit based on rules or heuristics, causal attribution uses scientific methods like the PC Algorithm, Propensity Score Matching, and Doubly Robust Estimation to discover true cause-and-effect relationships.

How is this different from Google Analytics attribution?

Google Analytics uses rule-based and data-driven attribution models that rely on correlation patterns. Our API uses causal inference techniques from academic research to identify true causal relationships, accounting for confounding variables and selection bias. This provides more accurate ROI measurements and better budget allocation decisions.

Do I need a data science background to use this API?

No! While the underlying methods are sophisticated, the API is designed to be simple to use. You send touchpoint and conversion data via JSON, and receive easy-to-interpret attribution scores and ROI metrics. We also provide examples in multiple programming languages and AI agent integrations for natural language queries.

Is this suitable for small businesses?

Absolutely! Our Starter plan at $99/month is designed for small businesses and startups. You get access to the same scientifically rigorous attribution models used by enterprise companies, at a fraction of the cost of competitors.

Attribution Models

What attribution models are available?

We offer 7+ attribution models:

  • data_driven - Our proprietary causal inference model (recommended)
  • shapley - Shapley Value attribution from game theory
  • first_touch - All credit to first touchpoint
  • last_touch - All credit to last touchpoint
  • linear - Equal credit to all touchpoints
  • time_decay - More credit to recent touchpoints
  • position_based - 40% first, 40% last, 20% middle touchpoints

Which model should I use?

We recommend starting with the data_driven model, which uses causal inference to discover true cause-and-effect relationships. For comparison purposes, you can also run shapley or traditional models to see how much they differ from causal attribution.

What is the PC Algorithm?

The PC (Peter-Clark) Algorithm is a constraint-based causal discovery algorithm that identifies causal relationships from observational data. It builds a directed acyclic graph (DAG) showing which touchpoints causally influence others and the conversion.

What is Propensity Score Matching?

Propensity Score Matching accounts for selection bias by comparing customers who were exposed to a channel with similar customers who weren't. This helps isolate the true causal effect of each channel.

What is Doubly Robust Estimation?

Doubly Robust Estimation combines outcome regression and propensity score weighting for more robust causal effect estimates. It provides accurate results even if one of the two models is misspecified.

How accurate is causal attribution?

Causal attribution is significantly more accurate than correlation-based methods. Studies show traditional attribution can misattribute up to 60% of credit. Our causal methods, when properly validated, achieve 85-95% accuracy in controlled experiments.

Data Requirements

How much data do I need?

You can analyze individual customer journeys with as few as 2 touchpoints. For the most accurate causal discovery, we recommend:

  • Minimum 100 completed customer journeys
  • At least 3-5 different marketing channels
  • Mix of converted and non-converted journeys (for some advanced models)

What data do I need to provide?

For basic attribution, you need:

  • Touchpoints: timestamp, channel name, cost, user_id
  • Conversion: timestamp, value, user_id

Optional fields include campaign name, creative ID, device type, and custom metadata.

Can I analyze non-conversion journeys?

Yes! While basic attribution focuses on converted journeys, our advanced causal models can incorporate non-conversion data to better understand channel effectiveness and identify optimization opportunities.

How do I handle multi-device journeys?

Use a consistent user_id across devices (like a logged-in user ID or cross-device identifier). The API will analyze the full journey regardless of device switches.

What if I don't have cost data for all touchpoints?

Cost data is optional but recommended for ROI calculations. If you don't provide costs, the API will still calculate attribution credits, but ROI metrics will be unavailable.
